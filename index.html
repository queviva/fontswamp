Welcome to the Font Swamp
Click this sentence to see a single line of javascript that allows users to change fonts in a page to any font name, or url, they specify; try double-clicking with the control-key pressed to change to large title above.

Include the fontswamp.js script, then create individual style tags for all the fonts you want to swamp, putting a data-font parameter inside the style tag.

Here's a sample of a default usage: 

<style data-font>
@font-face {
    font-family: mainPageFont;
    src: local('georgia');
}
</style>
<script src="fontswamp.js"></script>
By default, double clicking on any page element with a data-swamp parameter will launch a prompt, and overwrite the src of the font-face @rule in the style tag.
There can be several fontswamp style tags in a page, each with parameters specifying different options and behaviors. This code 

<style data-font='{
 "prompt" : "whachu fontin' about?!",
 "event" : "click",
 "usectrl" : true,
 "querylist" : ".someClass, .someOtherClass"
}'>
@font-face {
    font-family: mainPageFont;
    src: local('georgia');
}
</style>

<style data-font='{
 "usectrl" : true,
 "querylist" : ".header, [data-footer]"
}'>
@font-face {
    font-family: headFootFont;
    src: local('arial');
}
</style>
customizes the behavior so that the prompt will say "whachu fontin' about?", when you only single click, but are pressing the control key, on any page element with either the class "someClass" or "someOtherClass"

Any css tag with font-family: mainPageFont will change to the newly specified font.
Then, it creates another fontswamp that uses the default "enter a font name or url" prompt, when a default double click happens, while the control key is pressed, on any page elements with the class "header" or a data-footer parameter.

Any css tag with headFootFont in its font-family will get changed. Anywhere at all, actually, that the headFootFont family is mentioned will get changed.
